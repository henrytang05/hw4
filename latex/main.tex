\section{Introduction}

DQN is great for discrete action spaces. However, it requires calculating $\max_\rva Q(\rvs, \rva)$. Doing this is trivial for discrete action spaces, where you can exhaustively find which of the $n$ actions has the highest Q-value. However, in continuous action spaces, this can be a complex nonlinear optimization problem.

Actor-critic methods get around this by learning an explicit policy, $\pi$, that is trained to maximize $\mathbb{E}_{\rva \sim \pi(\rva \mid \rvs)} Q(\rvs, \rva)$. Throughout this assignment, you will learn to implement one of the most popular algorithms in this line, Soft Actor-Critic (SAC).

\end{lstlisting}

\section{Bootstrapping}

As in DQN, we train our critic by ``bootstrapping'' from a target critic. Using the tuple $(\rvs_t, \rva_t, r_t, \rvs_{t+1}, d_{t})$ (where $d_t$ is the flag for whether the trajectory terminates after this transition), we set a target value $y$:
\begin{equation}
    y \gets r_t + \gamma (1-d_t) Q_\phi(\rvs_{t+1}, \rva_{t+1})\text{, where } \rva_{t+1} \sim \pi(\rva_{t+1} \mid \rvs_{t+1}).
\end{equation}
Then, we use this bootstrapped target, $y$, to update our critic, $Q_\phi$, by minimizing the L2 loss:
\begin{equation}
    \min_\phi (Q_\phi(\rvs_t, \rva_t) - y)^2.
\end{equation}

In practice, we stabilize learning by using a separate \textit{target network}, $Q_{\phi'}$. There are two common strategies for updating the target network, as covered in lectures:
\begin{itemize}
    \item \textbf{Hard update} (implemented in DQN): set $\phi' \gets \phi$ every $K$ steps.
    \item \textbf{Soft update}: $\phi'$ is continually updated towards $\phi$ with \textit{Polyak averaging} (exponential moving average). After each update, we perform the following operation ($\tau$ is typically very small, e.g., $0.005$):
    \begin{equation}
        \phi' \gets \phi' + \tau(\phi-\phi').
    \end{equation}
\end{itemize}

\section{Entropy Bonus and Soft Actor-Critic}

In DQN, we use an $\epsilon$-greedy strategy to decide which action to take at a given time. In continuous spaces, we have several options for generating exploration noise.

One of the most common approaches is providing an \textit{entropy bonus} to encourage the actor to have high entropy (i.e. to be \textit{more random}), scaled by a ``\textbf{temperature}'' coefficient $\beta$. For example, in the REPARAMETRIZE case:
\begin{equation}
    \mathcal L_\pi = Q(\rvs, \mu_\theta(\rvs) + \sigma_\theta(\rvs)\epsilon) + \beta\mathcal{H}(\pi_\theta(\cdot \mid \rvs))\text{, where } \epsilon \sim \mathcal{N}(0, 1),
\end{equation}
where entropy is defined as $\mathcal H(\pi(\cdot \mid \rvs)) = \mathbb{E}_{\rva \sim \pi}\left[-\log\pi(\rva \mid \rvs)\right]$.

To make sure the entropy term is also factored into the Q-function, we use soft Q target values:
\begin{equation}
    y \gets r_t + \gamma(1-d_t)\left[Q_\phi(\rvs_{t+1}, \rva_{t+1}) + \beta\mathcal{H}(\pi(\cdot \mid \rvs_{t+1}))\right].
\end{equation}
This soft Q-values result in behaviors where the actor will choose more random actions when it is unsure of what action to take.
Please refer to the SAC paper: {\url{https://arxiv.org/abs/1801.01290}} for more details.

Note that maximizing entropy $\mathcal{H}(\pi_\theta) = -\mathbb{E}[\log \pi_\theta]$ requires differentiating \textit{through} the sampling distribution. We can do this via the ``reparametrization trick'' from lecture -- if you'd like a refresher, refer to the \Cref{sec:REPARAMETERIZE}.

\section{Actor Update with REINFORCE}

To train our actor to maximize rewards, we can use the REINFORCE gradient estimator that we used in Homework 2! 

The objective of the policy is to maximize the expected return and we need its gradient:
\begin{equation}
    \nabla_\theta\mathbb{E}_{\rvs \sim \mathcal{D}, \rva \sim \pi_\theta(\rva \mid \rvs)}\left[Q_\phi(\rvs, \rva)\right].
\end{equation}
As we learned in lecture and Homework 2, we can use REINFORCE to compute the policy gradient:
\begin{equation}
    \mathbb{E}_{\rvs \sim \mathcal{D}, \rva \sim \pi_\theta(\rva \mid \rvs)}\left[\nabla_\theta \log(\pi_\theta(\rva \mid \rvs))Q_\phi(\rvs, \rva)\right].
\end{equation}
Note that the actions $\rva$ are sampled from $\pi_\theta$ and we do \textbf{not} require real data samples. This means that to reduce the variance of this policy gradient, we can sample more actions from $\pi_\theta$ for any given state! You will implement this in your code using the \verb|num_actor_samples| parameter.

\section{Actor Update with REPARAMETRIZE}
\label{sec:REPARAMETERIZE}

REINFORCE works quite well with many samples, but particularly in high-dimensional action spaces, it starts to require a lot of samples to give low variance. We can improve this by using the reparametrized gradient. 

If we parameterize $\pi_\theta$ as $\mu_\theta(\rvs) + \sigma_\theta(\rvs)\epsilon$, where $\epsilon$ is normally distributed, then we can write:
\begin{equation}
    \nabla_\theta\mathbb{E}_{\rvs \sim \mathcal{D}, \rva \sim \pi_\theta(\rva|\rvs)}\left[Q(\rvs, a)\right] = \nabla_\theta\mathbb{E}_{\rvs \sim \mathcal{D}, \epsilon \sim \mathcal{N}}\left[Q(\rvs, \mu_\theta(\rvs) + \sigma_\theta(\rvs)\epsilon))\right] = \mathbb{E}_{\rvs \sim \mathcal{D}, \epsilon \sim \mathcal{N}}\left[\nabla_\theta Q(\rvs, \mu_\theta(\rvs) + \sigma_\theta(\rvs)\epsilon))\right].
\end{equation}
This gradient estimator often provides a much lower variance, so it can be used with few samples (in practice, just using a single sample works very well).

\section{Stabilizing Target Values}

As in DQN, the target $Q$ with a single critic exhibits \textit{overestimation bias}! There are a few commonly-used strategies to combat this:
\begin{itemize}
    \item \textbf{Double-Q}: learn two critics $Q_{\phi_A}, Q_{\phi_B}$, and keep two target networks $Q_{\phi_A'}, Q_{\phi_B'}$. Then, use $Q_{\phi'_A}$ to compute target values for $Q_{\phi_B}$ and vice versa (it's slightly different from Double DQN):
    \[y_A = r + \gamma Q_{\phi_B'}(\rvs', \rva')\]
    \[y_B = r + \gamma Q_{\phi_A'}(\rvs', \rva')\]
    
    \item \textbf{\textit{Clipped} double-Q}: learn two critics $Q_{\phi_A}, Q_{\phi_B}$ (and keep two target networks). Then, compute the target values as $\min(Q_{\phi_A'}, Q_{\phi_B'})$.
    \[y_A = y_B = r + \gamma \min\left(Q_{\phi_A'}(\rvs', \rva'), Q_{\phi_B'}(\rvs', \rva')\right)\]
\end{itemize}
